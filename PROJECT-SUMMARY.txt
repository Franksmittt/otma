1. PROJECT OVERVIEW
--------------------------------------------------------------------------------
• Name:          On The Move Again (OTMA)
• Purpose:       Marketing website for an Alberton-based furniture removals and
                 relocation company. Emphasis on secure, monitored moves (GPS
                 tracking), PMA & AMOSA accreditation, and full-service relocation.
• Base URL:      https://otma.co.za
• Tech stack:    Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS 4,
                 Geist font. No separate backend; quote/track are front-end flows.
• Key differentiator: “Smile! You’re on Camera!” – live GPS and in-transit
                 monitoring; Goods-in-Transit insurance; Joanique Eco Park base.

5. SEO IMPLEMENTATION
--------------------------------------------------------------------------------

5.1 Root metadata (layout.tsx)
  • title: default + template “%s | On The Move Again”
  • description: Alberton secure furniture removals, GPS, PMA/AMOSA, free quote.
  • keywords: furniture removals Alberton, movers Alberton, moving company Alberton,
    packers and movers, long distance movers Gauteng.
  • openGraph: title, description, url, siteName, locale (en_ZA).
  • twitter: card (summary_large_image), title, description.
  • alternates.canonical: SITE.url (homepage only; inner pages override – see 5.2).
  • robots: index: true, follow: true.

5.2 Per-page metadata and canonicals
  • Every page sets export const metadata (or generateMetadata for dynamic
    routes) with title and description.
  • Every page sets its own alternates.canonical to the full URL for that page
    (e.g. /about → https://otma.co.za/about, /blog/[slug] → https://otma.co.za/blog/{slug}).
    Homepage uses layout canonical (SITE.url).
  • Blog [slug]: generateMetadata with title, description, alternates.canonical,
    openGraph (article, publishedTime, url), twitter (card, title, description).
  • Suburb pages: generateMetadata from SUBURB_CONTENT with title, description,
    alternates.canonical.

5.3 JSON-LD structured data
  • MovingCompany (site-wide): In layout via <MovingCompanyJsonLd />.
    Includes: name, image, url, telephone, priceRange, address, geo,
    areaServed as Place[] with name + url (https://otma.co.za/movers-in/{slug})
    for each suburb (signals hyper-local landing pages), openingHoursSpecification,
    aggregateRating. File: src/components/JsonLd.tsx.
  • FAQPage: On /faq (full FAQ), and nested on key service pages for rich
    snippets: /insurance (3 FAQs: GIT vs Carrier’s Liability, coverage limit,
    trucks monitored), /services/house-removals (3 FAQs: monitored, areas, vehicle
    sizes). Each page has visible FAQ block + inline FAQPage JSON-LD.
  • BlogPosting: On every blog post (/blog/[slug]). headline, description,
    datePublished, author/publisher (Organization), mainEntityOfPage url.
    Inline in src/app/blog/[slug]/page.tsx (BlogPostingJsonLd).

5.4 Sitemap and robots
  • app/sitemap.ts: Generates XML sitemap at /sitemap.xml. Includes all static
    routes, all blog post slugs, all suburb slugs; lastModified, changeFrequency,
    priority per URL.
  • app/robots.ts: Allows all user agents, allow “/”, disallow none; sitemap
    URL set to https://otma.co.za/sitemap.xml.

5.5 Redirects (next.config.ts)
  • /movers-in-{suburb} → /movers-in/{suburb} (301) for:
    alberton, meyersdal, brackenhurst, new-redruth, randhart, alberante.
  • Ensures old “movers-in-alberton” style URLs still work.

5.6 HTML & accessibility
  • <html lang="en-ZA">.
  • Semantic structure: header, main, footer; article for blog posts.
  • Sticky CTA (Call / WhatsApp) on mobile only; focus and contrast tuned for
    Lighthouse (sky blue: text-sky-700, bg-sky-600, etc.).
  • Gallery: Placeholder blocks have descriptive aria-labels; comment in code
    instructs use of descriptive alt text when adding real photos.

5.7 Suburb SEO content
  • Each suburb has unique title, h1, description, and landmarks in
    src/lib/suburb-content.ts. Used by /movers-in/[slug] for metadata and
    on-page copy.
